Next Phase: Implementing Tinder-Style Matching Mechanism

Objective:

Build a matching mechanism that allows users to discover and engage with collaboration opportunities or event attendees based on their preferences. The experience should feel like a native telegram mini-app with an intuitive interface and seamless interaction using best in class UX and UI design.  

New Features to Implement:

1. New Bottom Navigation Bar (Mobile App Feel)

Discover (Icon: Two overlapping playing cards)

My Matches (Stores matched connections and Telegram integration)

Settings (No changes to the current dashboard, just housed under this tab)

2. Discover Section (Matching Process)

Users are shown collaboration or event opportunities as swipeable cards.

Each card displays a quick synopsis on the front:

Collaboration Type

User Job Title / Role at company  (found in the company database table)

Company name with link to their twitter 

Company sector tags

User interaction options:

❌ Pass (Skips to the next card)

**Details **(Flips the card to show detailed information about the collaboration and company - keep user info annonymous except for the Job Title / Role (found in the company database table))

✅ Request (Indicates interest in collaborating, marked by a tick)

Types of cards:

Marketing Collaboration Opportunities: Discover opportunities matching user preferences.

Conference Coffee Meets opportunities

Incoming Requests: Users see requests from others who want to collaborate with them.

Incoming Request cards should look slightly different to differentiate from standard opportunities.

Call these whatever you want in line with best app / copy practices

3. My Matches Section (Post-Match Actions)

Contact sharing: When both parties match for the collaboration, their details are added here.

List of matches - Name, Title, Company

More info button:

Telegram handle (primary communication method)

Personal Details

Company details

Collaboration details

Start Chat button: Clicking this opens a Telegram chat with the matched party.

All messaging happens in Telegram (No in-app chat feature).

4. User Anonymity & Privacy

Personal user information remains hidden until a match occurs.

Company information is publicly viewable.

5. UI & Experience Notes:

The dashboard layout remains unchanged, it is simply moved under the Settings tab.

The swiping/matching mechanism should be intuitive and visually engaging. I want the user to feel familiar about this from apps like tinder

When a user requests more info, the card should reveal details or go into a detailed view like you would on a dating app.

Subtle differentiation in card design between discovering collaborations vs. receiving collaboration requests.
